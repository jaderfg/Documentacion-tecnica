Introduction

JavaScript es un lenguaje de secuencias de comandos orientado a objetos y multiplataforma. Es un pequeño y ligero idioma. Dentro de un entorno de host (por ejemplo, un navegador web), JavaScript se puede Conectado al objetos de su entorno para proporcionar control programático sobre ellos.

JavaScript contiene una biblioteca estándar de objetos, como Array, Date y Math, y un núcleo conjunto de elementos del lenguaje como operadores, estructuras de control y declaraciones. Core JavaScript puede ser extendido para una variedad de propósitos completándolo con objetos adicionales; por ejemplo:

  • JavaScript del lado del cliente extiende el lenguaje central al proporcionar objetos para controlar un navegador y es Modelo de objetos de documento (DOM). Por ejemplo, las extensiones del lado del cliente permiten que una aplicación lugar elementos en un formulario HTML y responder a los eventos del usuario, como los clics del mouse, la entrada del formulario y la página navegación.
  • JavaScript del lado del servidor amplía el lenguaje central al proporcionar objetos relevantes para ejecutar JavaScript sobre un servidor. Por ejemplo, las extensiones del lado del servidor permiten que una aplicación se comunique con un base de datos, proporcionar continuidad de información de una invocación a otra de la aplicación, o realizar archivo manipulaciones en un servidor.

    • JavaScript_and_Java

    JavaScript y Java son similares en algunos aspectos, pero fundamentalmente diferentes en otros. los JavaScript El lenguaje se parece a Java, pero no tiene la escritura estática ni la verificación de tipo fuerte de Java. JavaScript sigue la mayoría de la sintaxis de expresión de Java, convenciones de nomenclatura y construcciones básicas de flujo de control que era los razón por la que se cambió el nombre de LiveScript a JavaScript.

    En contraste con el sistema de clases en tiempo de compilación de Java construido por declaraciones, JavaScript apoya un tiempo de ejecución sistema basado en una pequeña cantidad de tipos de datos que representan números, booleanos y cadenas valores. JavaScript tiene un modelo de objetos basado en prototipos en lugar del objeto basado en clases más común modelo. los el modelo basado en prototipos proporciona herencia dinámica; es decir, lo que se hereda puede variar para individual objetos. JavaScript también admite funciones sin requisitos declarativos especiales. Funciones pueden ser propiedades de los objetos, ejecutándose como métodos de tipo impreciso.

    JavaScript es un lenguaje de forma muy libre en comparación con Java. No tienes que declarar todo variables, clases y métodos. No tiene que preocuparse por si los métodos son públicos, privado, o protegido, y no tiene que implementar interfaces. Variables, parámetros y función regreso tipos no se escriben explícitamente.

    Variables

    Utiliza variables como nombres simbólicos para valores en su aplicación. Los nombres de las variables, llamado identificadores, se ajustan a ciertas reglas.

    Un identificador de JavaScript debe comenzar con una letra, un guión bajo (_) o un signo de dólar ($); subsecuente caracteres también pueden ser dígitos (0-9). Debido a que JavaScript distingue entre mayúsculas y minúsculas, las letras incluyen los caracteres "A" mediante "Z" (mayúsculas) y los caracteres "a" a "z" (minúsculas).

    Puede utilizar letras ISO 8859-1 o Unicode como å y ü en los identificadores. También puede utilizar el Unicode secuencias de escape como caracteres en identificadores. Algunos ejemplos de nombres legales son Number_hits, temp99 y _name.

    Declaring_variables
    Puede declarar una variable de tres formas:

    Con la palabra clave var. Por ejemplo, var x = 42. Esta sintaxis se puede utilizar para declarar ambos variables locales y globales.

    Simplemente asignándole un valor. Por ejemplo, x = 42. Esto siempre declara un global variable. Genera una advertencia estricta de JavaScript. No deberías usar esta variante.

    Con la palabra clave dejar. Por ejemplo, let y = 13. Esta sintaxis se puede utilizar para declarar un cuadra ámbito de la variable local. Consulte Alcance variable a continuación.

    Variable_scope

    Cuando declaras una variable fuera de cualquier función, se llama variable global, porque está disponible para cualquier otro código en el documento actual. Cuando declaras una variable dentro de un función, eso se llama variable local, porque solo está disponible dentro de esa función.

    JavaScript antes de ECMAScript 2015 no tiene alcance de declaración de bloque; más bien, una variable declarado dentro de un bloque es local a la función (o ámbito global) en el que reside el bloque. Para ejemplo el el siguiente código registrará 5, porque el alcance de x es la función (o contexto global) dentro de cual x es declarado, no el bloque, que en este caso es una sentencia if.

    if (true) { var x = 5; } console.log(x); // 5

    Este comportamiento cambia cuando se usa la declaración let introducida en ECMAScript 2015.

    if (true) { let y = 5; } console.log(y); // Error de referencia: y no está definida
    Global_variables

    Las variables globales son de hecho propiedades del objeto global. En las páginas web el objeto global es ventana, para que pueda establecer y acceder a las variables globales utilizando la sintaxis window.variable.

    En consecuencia, puede acceder a las variables globales declaradas en una ventana o marco desde otra ventana o frame especificando el nombre de la ventana o del marco. Por ejemplo, si una variable llamada phoneNumber es declarado en un documento, puede hacer referencia a esta variable desde un iframe como parent.phoneNumber.

    Constants

    Puede crear una constante de solo lectura con nombre con la palabra clave const. La sintaxis de una constante el identificador es lo mismo que para un identificador de variable: debe comenzar con una letra, un guión bajo o un dólar firmar y pueden contener caracteres alfabéticos, numéricos o de subrayado.

    const PI = 3.14;

    Una constante no puede cambiar el valor a través de la asignación o volver a declararse mientras el script está corriendo. Eso tiene para ser inicializado a un valor.

    Las reglas de alcance para las constantes son las mismas que las de las variables de alcance de bloque let. Si el constante palabra clave se omite, se supone que el identificador representa una variable.

    No puede declarar una constante con el mismo nombre que una función o variable en el mismo ámbito. Por ejemplo:

    // ESTO CAUSARÁ UN ERROR function f() {}; const f = 5; // ESTO CAUSARÁ UN ERROR ADEMAS function f() { const g = 5; var g; //declaraciones } Sin embargo, los atributos del objeto no están protegidos, por lo que la siguiente instrucción se ejecuta sin problemas. const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";
    Data_types

    El último estándar ECMAScript define siete tipos de datos:

  • Seis tipos de datos que son primitivos:

    • Boolean. verdadero y falso.
    • null. Una palabra clave especial que denota un valor nulo. Dado que JavaScript distingue entre mayúsculas y minúsculas, nulo es no lo mismo que Null, NULL o cualquier otra variante.
    • undefined. Una propiedad de nivel superior cuyo valor no está definido.
    • Number. 42 o 3.14159.
    • String. "Hola"
    • Símbolo (nuevo en ECMAScript 2015). Un tipo de datos cuyas instancias son únicas y inmutable.
  • Object
    • Aunque estos tipos de datos son una cantidad relativamente pequeña, le permiten realizar funciones con sus aplicaciones. Los objetos y las funciones son los otros elementos fundamentales del lenguaje. usted pueden pensar de objetos como contenedores con nombre para valores y funciones como procedimientos que su aplicación puede llevar a cabo.
    if...else_statement
    Utilice la instrucción (if) para ejecutar una instrucción si una condición lógica es verdadera. Utilice el opcional else cláusula para ejecutar una declaración si la condición es falsa. Una declaración if tiene el siguiente aspecto: if (condicion) { statement_1; } else { statement_2; } La condición puede ser cualquier expresión que se evalúe como verdadera o falsa. Consulte booleano para obtener una explicación. de que evalúa verdadero y falso. Si la condición se evalúa como verdadera, se ejecuta statement_1; de lo contrario, declaración_2 es ejecutado. declaración_1 y declaración_2 pueden ser cualquier declaración, incluso anidadas si declaraciones.

    También puede componer las declaraciones usando else if para tener múltiples condiciones probadas en secuencia, como sigue:

    if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; } En el caso de múltiples condiciones, solo la primera condición lógica que se evalúa como verdadera será ser ejecutado. Para ejecutar varias sentencias, agrúpelas dentro de una sentencia de bloque ({...}). En general, es buena práctica para usar siempre sentencias de bloque, especialmente al anidar sentencias if: if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; } Es aconsejable no utilizar asignaciones simples en una expresión condicional, porque el la asignación puede ser confundido con igualdad al mirar el código. Por ejemplo, no utilice el siguiente código: if (x = y) { /* statements here */ } Si necesita usar una asignación en una expresión condicional, una práctica común es poner adicional paréntesis alrededor de la tarea. Por ejemplo: if ((x = y)) { /* statements here */ }
    while_statement
    Una instrucción while ejecuta sus declaraciones siempre que una condición especificada se evalúe como verdadera. A tiempo declaración tiene el siguiente aspecto: while (condition) statement Si la condición se vuelve falsa, la declaración dentro del ciclo deja de ejecutarse y el control pasa a los declaración siguiendo el circuito.

    La prueba de condición ocurre antes de que se ejecute la instrucción en el ciclo. Si la condición regresa cierto, se ejecuta la sentencia y la condición se prueba de nuevo. Si la condición devuelve falso, ejecución se detiene y el control se pasa a la instrucción que sigue al circuito.

    Para ejecutar varias declaraciones, use una declaración de bloque ({...}) para agrupar las declaraciones.

    Ejemplo:

    El siguiente ciclo while se repite siempre que sea menor que tres:

    var n = 0; var x = 0; while (n < 3) { n++; x += n; }

    Con cada iteración, el ciclo incrementa n y agrega ese valor a x. Por lo tanto, x y n toman sobre el siguientes valores:

  • Después del primer paso: n = 1 and x = 1
  • Después del segundo paso: n = 2 and x = 3
  • Después del tercer paso: n = 3 and x = 6

    • Después de completar la tercera pasada, la condición n & lt 3 ya no es cierta, por lo que el circulo termina.

    Function_declarations
    Una definición de función (también llamada declaración de función o declaración de función) consta de los función palabra clave, seguida de:
  • El nombre de la función.
  • Una lista de argumentos de la función, entre paréntesis y separados por comas.
  • Las declaraciones de JavaScript que definen la función, entre llaves, {}.

    • Por ejemplo, el siguiente código define una función simple llamada cuadrado:

    function square(number) { return number * number; }

    La función cuadrado toma un argumento, llamado número. La función consta de una declaración ese dice para devolver el argumento de la función (es decir, número) multiplicado por sí mismo. El regreso declaración especifica el valor devuelto por la función.

    return number * number;

    Los parámetros primitivos (como un número) se pasan a las funciones por valor; se pasa el valor al función, pero si la función cambia el valor del parámetro, este cambio no es reflejado globalmente en la funcion declarada
